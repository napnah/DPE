# P4 — Yjs Secure Sync (SignedUpdate + Merge Guards)

## Scope

| Package | Deliverable |
|---------|-------------|
| `@dpe/yjs-provider` | `SecureYjsProvider`: Yjs updates ↔ SignedUpdate, §3.5 merge前校验, role-1 拒写 |

Builds on P3 authenticated DataChannels; does not include full browser WebRTC mesh (P5 UI).

## Usage

```typescript
import * as Y from "yjs";
import { SecureYjsProvider } from "@dpe/yjs-provider";

const doc = new Y.Doc();
const provider = new SecureYjsProvider({
  doc,
  docId: "root",
  local: {
    nodeId,
    role: 2,
    privateKey,
    publicKey,
    docKey,       // opened from JWT doc_key
    keyVersion: 1,
  },
  send: (frame) => dataChannel.send(frame),
});

provider.registerPeer(peerSessionAfterAuth);
provider.bindChannel(dataChannel); // handles { type: "sync", update } frames
```

## Merge policy (方案 §3.5)

Incoming `SignedUpdate` is applied only when:

1. `signer_node_id` is a registered peer with `role >= 1` (receive)
2. `key_version` matches local
3. Ed25519 signature valid and `sub` matches signer
4. **`role >= 2`** for content writes (readonly sends are rejected, not silently dropped)
5. `(signer, seq)` not in replay cache
6. AES-GCM decrypt → `Y.applyUpdate`

Local outbound updates require `local.role >= 2`.

## Wire frame

```json
{ "type": "sync", "update": { "doc_id", "key_version", "nonce", "ciphertext", "signer_node_id", "seq", "sig" } }
```

Sent on the DataChannel after AuthEnvelope (P3).
