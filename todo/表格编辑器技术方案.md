# 表格编辑器技术方案

## 1. 背景

当前 DPE 已经具备多数据格式编辑的基础：

- 每个文档仍然对应一个 `Y.Doc`。
- `content` 保留为兼容旧文档的主文本字段。
- `meta.editorMode` 用于在纯文本、Markdown、块文档等编辑模式之间切换。
- `blocks` 用于保存结构化块。
- `assets` 用于保存图片等资源元数据。
- 本地持久化和远端快照都保存完整 Yjs state。
- P2P 同步通过 `SecureYjsProvider` 监听整个 `Y.Doc` 的 update，不关心 update 来自文本、Map、Array 还是其他 Yjs 类型。

因此，表格能力不需要改动底层同步、安全协议、JWT、ACL 或快照接口。需要设计的是前端数据模型和编辑界面。

本方案把表格分成两类：

- 独立表格文件：一个文档就是一个表格，适合演示类 Excel / CSV 的独立数据文件。
- 块文档表格块：表格作为块文档中的一种 block，和段落、Markdown、图片、代码、Todo 混排。

## 2. 目标

### 2.1 独立表格文件

提供一个专门的表格编辑模式，让一个文档可以作为完整表格文件使用：

- 支持单元格编辑。
- 支持新增 / 删除行列。
- 支持行列标题显示。
- 支持基础数据类型：文本、数字、布尔值。
- 支持简单 CSV 导入 / 导出。
- 支持通过 Yjs 同步和快照保存。

### 2.2 块文档表格块

在块文档模式中新增 `table` block：

- 一个块内展示一个轻量表格。
- 支持编辑单元格。
- 支持新增 / 删除行列。
- 支持表格标题或说明。
- 支持在文档中与图片、Markdown、代码、任务项混排。

## 3. 非目标

第一阶段不做完整电子表格软件能力：

- 不做公式计算引擎。
- 不做跨 sheet 引用。
- 不做复杂单元格格式、条件格式、冻结窗格。
- 不做图表。
- 不做多人光标和选区同步。
- 不做大规模表格虚拟滚动，第一阶段面向中小型演示表格。

如果后续要支持公式和大表，应单独引入表格内核或专门做增量架构升级。

## 4. 数据模型

### 4.1 meta 扩展

现有 `meta.editorMode` 建议扩展为：

```ts
type EditorMode = "plain" | "markdown" | "blocks" | "table";
```

独立表格文件使用：

```ts
ymeta.set("editorMode", "table");
ymeta.set("schemaVersion", 2);
```

老文档兼容策略不变：

- 没有 `meta.editorMode` 时仍按 `plain` 处理。
- `table` 模式只在用户显式切换或创建表格文档时启用。

### 4.2 独立表格文件结构

建议新增一个 Yjs 类型：

```ts
const ytable = doc.getMap("table");
```

表格元数据：

```ts
type TableMeta = {
  id: string;
  rowCount: number;
  columnCount: number;
  columns: TableColumn[];
  createdAt?: string;
  updatedAt?: string;
};

type TableColumn = {
  id: string;
  name: string;
  type: "text" | "number" | "boolean";
  width?: number;
};
```

单元格数据建议先用普通 JSON 存储：

```ts
type TableCellValue = string | number | boolean | null;

type TableData = {
  rows: TableCellValue[][];
};
```

第一阶段可以放在：

```ts
ytable.set("meta", tableMeta);
ytable.set("data", tableData);
```

优点：

- 实现简单。
- 快照和 P2P 能自然保存。
- 中小表格演示足够。

缺点：

- 并发编辑粒度较粗，多个用户同时改同一张表时可能出现整块替换冲突。

长期优化可以改成更细粒度结构：

```ts
const ytableRows = doc.getArray("tableRows");
// 每行是 Y.Map，每个单元格是 key/value。
```

但第一阶段不建议过早复杂化。

### 4.3 块文档表格块结构

在 `DocBlock` 中新增：

```ts
type TableBlock = {
  id: string;
  type: "table";
  caption?: string;
  columns: TableColumn[];
  rows: TableCellValue[][];
};
```

完整块类型变为：

```ts
type DocBlock =
  | { id: string; type: "paragraph"; text: string }
  | { id: string; type: "markdown"; source: string }
  | { id: string; type: "image"; assetId: string; alt?: string; caption?: string }
  | { id: string; type: "code"; language?: string; code: string }
  | { id: string; type: "todo"; checked: boolean; text: string }
  | TableBlock;
```

块表格不需要独立的 `Y.Map("table")`。它作为 `Y.Array("blocks")` 中的一个 JSON block 保存。

## 5. UI 设计

### 5.1 独立表格文件界面

新增 `TableSurface`：

```text
DocInlineEditor
  ├─ PlainTextSurface
  ├─ MarkdownSurface
  ├─ BlockDocumentSurface
  └─ TableSurface
```

顶部工具栏：

- 模式切换：纯文本 / Markdown / 块文档 / 表格。
- 表格操作：新增行、新增列、删除选中行、删除选中列。
- 导入导出：导入 CSV、导出 CSV。
- 状态展示：只读、保存状态、快照大小提示。

表格主体：

- 第一行显示列名。
- 左侧显示行号。
- 单元格使用 `<input>` 或轻量 contenteditable。
- 数字列输入时做基础校验。
- 布尔列用 checkbox。
- 列名可编辑。

只读规则：

- `role < 2` 时，单元格、列名、行列操作、导入按钮全部 disabled。
- 只读用户仍可复制单元格内容和导出 CSV。

### 5.2 块文档表格块界面

在 `BlockDocumentSurface` 的插入工具栏中新增“表格”按钮。

表格块展示：

- 块顶部显示类型 `table`、上移、下移、删除。
- 支持 caption 输入。
- 表格内支持编辑列名和单元格。
- 支持新增行、新增列。
- 支持删除最后一行 / 最后一列，第一阶段不做复杂选区删除。

建议默认插入：

```ts
{
  id: createBlockId(),
  type: "table",
  caption: "",
  columns: [
    { id: "col_a", name: "A", type: "text" },
    { id: "col_b", name: "B", type: "text" }
  ],
  rows: [
    ["", ""],
    ["", ""]
  ]
}
```

## 6. CSV 导入导出

### 6.1 独立表格文件

第一阶段只要求独立表格文件支持 CSV 导入 / 导出。

导入流程：

1. 用户点击“导入 CSV”。
2. 选择 `.csv` 文件。
3. 前端读取文本。
4. 使用轻量 CSV parser 解析。
5. 第一行默认作为列名。
6. 写入 `Y.Map("table")`。

导出流程：

1. 读取 `table.meta.columns` 和 `table.data.rows`。
2. 生成 CSV 字符串。
3. 使用 Blob 下载。

注意事项：

- CSV parser 需要支持双引号转义。
- 第一阶段不处理复杂编码识别，默认 UTF-8。
- 导入时限制最大行列规模，例如 200 行、30 列。

### 6.2 块文档表格块

块表格第一阶段不做 CSV 文件导入导出。

原因：

- 块表格主要用于文档内混排演示。
- 导入导出会增加块级操作复杂度。
- 独立表格文件已经覆盖“文件级表格”场景。

后续如果需要，可以在单个表格块右上角增加“复制为 CSV”。

## 7. 同步与持久化

### 7.1 P2P 同步

不需要修改 `SecureYjsProvider`：

- 独立表格文件写入同一个 `Y.Doc`。
- 块表格写入 `Y.Array("blocks")`。
- provider 监听完整 Yjs update，仍然加密、签名并做权限校验。

### 7.2 本地持久化

不需要修改 `doc-persistence.ts` 的核心逻辑：

- `Y.encodeStateAsUpdate(doc)` 会包含 `table`、`blocks`、`meta`。
- localStorage 保存完整 state。

建议新增测试：

- 保存并恢复 `meta.editorMode = "table"`。
- 保存并恢复 `Y.Map("table")`。
- 保存并恢复 `blocks` 中的 `table` block。

### 7.3 远端快照

接口不变，仍使用：

```text
state_update_base64
```

但需要前端控制表格规模：

- 独立表格文件第一阶段建议限制 200 行、30 列。
- 块表格建议限制 50 行、12 列。
- 保存前继续计算 `Y.encodeStateAsUpdate(doc).length`。
- 超过 1.5MB 时提示用户表格过大，远端快照可能失败。

## 8. 实施计划

### 阶段一：块文档表格块

目标：最快在现有块文档中展示“多数据格式混排 + 表格”。

改动范围：

- 扩展 `DocBlock`，新增 `table` 类型。
- `BlockDocumentSurface` 新增“表格”插入按钮。
- 新增 `TableBlockEditor`。
- 支持编辑 caption、列名、单元格。
- 支持新增行、新增列、删除末行、删除末列。
- 只读用户不能编辑。

验收：

- 块文档中可插入表格。
- 表格可以和段落、Markdown、图片、代码、Todo 混排。
- 刷新后表格仍存在。
- 另一浏览器实例能同步看到表格变化。
- 只读用户不能修改表格。

### 阶段二：独立表格文件模式

目标：让一个文档可以作为完整表格文件编辑。

改动范围：

- 扩展 `EditorMode`，新增 `"table"`。
- 新增 `TableSurface`。
- 新增 `Y.Map("table")` 数据读写逻辑。
- 顶部模式切换增加“表格”。
- 支持新增 / 删除行列。
- 支持 CSV 导入 / 导出。
- 保存前增加表格规模和快照大小提示。

验收：

- 可切换到表格模式。
- 可编辑单元格和列名。
- 可新增 / 删除行列。
- 可导入 CSV。
- 可导出 CSV。
- 刷新和 P2P 同步后表格内容一致。

### 阶段三：表格体验增强

目标：提升演示观感和基础可用性。

可选能力：

- 单元格键盘导航：Enter、Tab、方向键。
- 粘贴 TSV / Excel 区域数据。
- 简单列类型识别。
- 列宽拖动。
- 固定表头。
- 当前单元格高亮。
- 复制选中区域为 TSV。

### 阶段四：长期表格内核升级

如果后续要支持大表和多人高频并发编辑，建议升级数据模型：

- 从普通 JSON 整表替换，升级为行级或单元格级 Yjs 类型。
- 每行使用 `Y.Map`。
- 单元格值按列 id 存储。
- 增加虚拟滚动。
- 考虑引入成熟表格渲染库，但仍保持 Yjs 数据层可控。

## 9. 风险与处理

| 风险 | 说明 | 处理 |
|------|------|------|
| 表格 JSON 整体替换冲突 | 多人同时编辑同一张表，最后写入可能覆盖部分内容 | 第一阶段接受；后续升级为行级 / 单元格级 Yjs 类型 |
| 快照过大 | 大表会推高 `state_update_base64` 大小 | 限制行列规模，保存前检查 state 大小 |
| UI 复杂度上升 | 表格编辑控件会让 `DocInlineEditor` 更大 | 抽出 `TableSurface`、`TableBlockEditor`、表格工具函数 |
| CSV 解析边界 | 引号、换行、逗号转义容易出错 | 第一阶段实现受限 parser 并写测试；复杂场景后续引入库 |
| 只读权限遗漏 | 表格有多个入口可能漏禁用 | 所有写操作统一检查 `writable` |

## 10. 推荐落地顺序

推荐先做块文档表格块，再做独立表格文件：

1. 块文档表格块改动小，直接复用现有 `blocks` 结构，演示效果明显。
2. 独立表格文件需要新增完整 `TableSurface` 和 CSV 能力，适合作为第二阶段。
3. 后续再补键盘导航、粘贴、列宽等体验增强。

最终目标结构：

```text
DocInlineEditor
  ├─ PlainTextSurface
  ├─ MarkdownSurface
  ├─ BlockDocumentSurface
  │    ├─ ParagraphBlock
  │    ├─ MarkdownBlock
  │    ├─ ImageBlock
  │    ├─ CodeBlock
  │    ├─ TodoBlock
  │    └─ TableBlock
  └─ TableSurface
       ├─ TableToolbar
       ├─ TableGrid
       └─ CsvImportExport
```

这样既能满足“一个专门的表格文件”，也能满足“块文档里插入表格”的混排场景，并且不破坏现有加密同步、快照持久化和权限校验链路。
