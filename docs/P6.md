# P6 — E2E、安全硬化与双平台验收

## Scope

| 项 | 说明 |
|----|------|
| **全量验收** | `pnpm verify:p6`：P5 + 全包 `test`/`lint` + 威胁模型静态审计 |
| **E2E 冒烟** | `pnpm verify:p6 --live`：Postgres + 四服务 + 群组/邀请/文档/ACL/JWT/树 API 链路 |
| **安全审计** | `node scripts/security-audit.mjs`：JWT 封装、合并前校验、AuthEnvelope、Web 不 stale-verify |
| **双机验收** | [acceptance-dual-host.md](./acceptance-dual-host.md)（Windows + Linux 手工清单） |
| **CI** | GitHub Actions：两个 job — `ubuntu` + Postgres 跑 `--live`；`windows` 跑离线 `verify:p6`（无 service 容器） |

## Verify

```bash
# 离线（无需数据库，适合 Windows CI / 本地快验）
pnpm verify:p6

# 完整 E2E（需 Postgres；含 P5 基线 + test + lint）
docker compose up -d postgres
cd apps/control-plane && pnpm db:push && cd ../..
pnpm verify:p6 --live

# 仅 E2E 栈（跳过 P1–P5 重复构建，开发调试用）
pnpm verify:p6:live
```

单独跑 API 冒烟（控制平面已启动，默认 `http://127.0.0.1:3001`）：

```bash
node scripts/e2e-smoke.mjs
# 或指定端口（verify:p6 --live 使用 3106）
node -e "import('./scripts/e2e-smoke.mjs').then(m=>m.runE2eSmoke({controlPlaneUrl:'http://127.0.0.1:3106'}))"
```

`--live` 使用端口 **3106 / 3105 / 3104**，避免与 `pnpm dev`（3001–3003）冲突。

## E2E 覆盖（`e2e-smoke.mjs`）

1. 建群（owner）
2. 邀请 → 成员接受（root 只读 ACL）
3. `CreateChild` 在 `root` 下新建文档
4. `SetDocRoleAcl` 将「读者」角色在该子文档上设为可写（入群成员默认读者角色）
5. 成员 `jwt/refresh`：`role=2`、密封 `doc_key`
6. 成员 `tree`：可见 `root`（`isFolder`）与子文档
7. 成员在无 Operable 时 `CreateChild` 被拒绝
8. `GET /members` 至少 2 人

`--live` 额外：信令 WS join/peers、lan-agent 手动 peer 注册与搜索。

## 威胁模型对照（方案 §3.5 / §7）

| 检查 | 实现位置 |
|------|----------|
| `doc_key` 密封下发 | `SigningService.sealDocKeyForMember` |
| 合并前 role / 签名 / 重放 | `@dpe/yjs-provider` `merge-guard.ts` |
| 只读 peer 写更新拒绝 | `secure-provider.test.ts` |
| P2P 首包 AuthEnvelope | `@dpe/p2p` |
| 编辑器信任 refresh 后解析 JWT | `DocEditorPage` + `parseJwtPayload` |
| root 为目录非文档 | `ROOT_DOC_ID` + `isFolder` |

## 双平台

- **CI**：`ubuntu-latest` + `windows-latest` 矩阵（见 `.github/workflows/ci.yml`）
- **手工双机**：两台 LAN 主机各跑 `pnpm dev`，按 [acceptance-dual-host.md](./acceptance-dual-host.md) 勾项

## 已知局限（方案诚实说明）

- 恶意或沦陷的 **Issuer/Proxy** 可读全群明文；E2E 不替代渗透测试。
- RBAC 三表（角色列表 / 映射 / 默认规则）已入库；E2E 使用 `SetDocRoleAcl` + `GET /governance` 解析角色 ID。
